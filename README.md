# Платформа управления заказами

Простая веб-платформа для управления заказами, готовая к размещению на GitHub Pages.

## Функционал

- ✅ Добавление новых заказов (номер, имя исполнителя, статус)
- ✅ Редактирование существующих заказов
- ✅ Удаление заказов
- ✅ Поиск заказов по номеру, имени или статусу
- ✅ Автоматическое сохранение данных в браузере (localStorage)
- ✅ Адаптивный дизайн для мобильных устройств

## Как запустить локально

### Способ 1: Просто открыть в браузере (самый простой)

1. Найдите файл `index.html` в папке проекта
2. Дважды кликните на него или откройте через браузер
3. Готово! Приложение откроется в браузере

**Примечание:** При таком способе адрес будет начинаться с `file://`, что нормально для этого проекта.

### Способ 2: Через локальный HTTP сервер (рекомендуется)

#### Вариант A: Использовать готовый скрипт (Windows)

1. Дважды кликните на файл `start-server.bat`
2. Откройте в браузере: `http://localhost:8000`
3. Для остановки сервера нажмите `Ctrl+C` в окне терминала

#### Вариант B: Через Python (если установлен)

Откройте терминал в папке проекта и выполните:

```bash
# Python 3
python -m http.server 8000

# или Python 2
python -m SimpleHTTPServer 8000
```

Затем откройте в браузере: `http://localhost:8000`

#### Вариант C: Через Node.js (если установлен)

Установите простой сервер:
```bash
npm install -g http-server
```

Затем запустите:
```bash
http-server -p 8000
```

Откройте в браузере: `http://localhost:8000`

#### Вариант D: Через VS Code

Если используете VS Code:
1. Установите расширение "Live Server"
2. Правой кнопкой на `index.html` → "Open with Live Server"
3. Автоматически откроется в браузере

## Как разместить на GitHub Pages

### Вариант 1: Через настройки репозитория (рекомендуется)

1. Создайте новый репозиторий на GitHub
2. Загрузите все файлы проекта в репозиторий
3. Перейдите в **Settings** → **Pages**
4. В разделе **Source** выберите ветку `main` (или `master`)
5. Выберите папку `/ (root)`
6. Нажмите **Save**
7. Ваш сайт будет доступен по адресу: `https://ваш-username.github.io/название-репозитория/`

### Вариант 2: Через GitHub CLI

```bash
# Инициализация git репозитория
git init
git add .
git commit -m "Initial commit"

# Добавление удаленного репозитория
git remote add origin https://github.com/ваш-username/название-репозитория.git
git branch -M main
git push -u origin main
```

Затем следуйте инструкциям из Варианта 1, начиная с шага 3.

## Технологии

- HTML5
- CSS3 (с градиентами и анимациями)
- Vanilla JavaScript (без зависимостей)
- LocalStorage для хранения данных

## Особенности

- **Без сервера**: Все данные хранятся локально в браузере
- **Быстрая работа**: Нет загрузки внешних библиотек
- **Простота**: Легко настроить и кастомизировать
- **Офлайн работа**: Приложение работает без интернета

## Кастомизация

Вы можете легко изменить:
- Цвета в файле `styles.css` (градиенты, кнопки, статусы)
- Статусы заказов в `index.html` и `script.js`
- Стили карточек заказов в `styles.css`

## Примечания

- Данные хранятся в localStorage браузера, поэтому они будут доступны только на том устройстве, где вы их создали
- Для синхронизации между устройствами потребуется бэкенд (не входит в этот проект)
